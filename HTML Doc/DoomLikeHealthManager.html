Doom-Like Health Manager

Written by Kevin Boylan, C00192781

 

Topics

Creating a Doom-like Health Manager using a Model View Controller. 2 different views will be updated every time the health changes.

Example

  • We will enter a Health value into a text box.
  • healthManager.update() is called and a Health Bar view (a traditional Health Bar) and a Face view (outputs a particular image based on how low or high your health is) are both updated automatically.

Objective

  • This learning resource is designed to help users understand how to create a Health Manager for any Javascript game.
  • Users will also gain an understanding of some of the concepts and difficulties associated with Model View Controllers.
  • Javascript projects are often structured poorly and suffer from “spaghetti code”. By incorporating the Model View Controller framework, programmers will have neater and much more reusable code.

Prerequisites

  • Users will need to know how to set up a canvas and a context.
  • Users will need a basic knowledge of computer graphics programs. In this case, Paint.Net will be required to create a sprite sheet for the DoomFaceView.

Model View Controller

  • Before proceeding forward, it’s important that we first understand the Model View Controller.
  • The concept of the Model View Controller is often very poorly explained, but it is simpler than it seems.

Model

  • Contains all variables and logic.  
  • The Model passes all of this data to the View.

View

  • Renders the Model into a form suitable for viewing.
  • In short, this is the display. The View is passed to the Controller.

Controller

  • Processes and responds to events, typically user actions, and invokes changes on the Model.
  • In short, it changes changes/updates the Model.

In short, the Model updates the View, the View updates the Controller and the Controller updates the View.

4. Creating the View(s)

  • We have 2 different Views.
  • The Views are listening to the Controller.
  • In Main.cpp, we instantiate both Views and the Controller is passed in as a parameter for each of them:

HealthBarView

  • The HealthBarView is a traditional Health Bar.
  • Health can be some value between 0 (Minimum Health) and 100 (Maximum Health).
  • We handle the health in our update
  • The Bar will adjust according to our Health (e.g. 65) relative to our Maximum Health.
  • We also display the Health value through text. The font is optional.

DoomFaceView

  • Our DoomFaceView will be somewhat similar to our HealthBarView.
